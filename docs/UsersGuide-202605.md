# PointMarker 利用者の手引

**Version 2026年5月版 v6.2**

---

## はじめに

PointMarkerは、ハイキングマップ画像（PNG形式）上にポイント・スポット・ルート・エリアを直感的にマーキングし、JSONデータとして保存・管理するWebアプリケーションです。

### v6.2 の変更点

- **コントロールパネルがドラッグで動かせるようになりました**: パネル上部のタイトル部分（薄いベージュの帯）をマウスでドラッグして、画像の見たい部分が隠れない場所へ自由に移動できます
- **ズーム・移動した状態では、読み込み・保存・出力ができなくなりました**: 操作ミスを防ぐため、画像を元の表示状態に戻してから実行する仕様に変更しました。該当時は警告メッセージで案内されます
- **データ操作中の進捗が表示されるようになりました**: DBの読み込み・保存中に「ポイントのデータを読み込み中...」など、現在処理中のデータ種別が分かるメッセージが表示されます。完了時には種別ごとの件数サマリーが表示されます

---

## 1. 起動方法

PointMarkerはローカルサーバー経由で起動する必要があります。

```bash
# PointMarkerフォルダで実行
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いてください。

> **注意**: ファイルを直接ブラウザで開く（`file://`）と動作しません。

---

## 2. 基本的な使い方

### ステップ1: 画像を読み込む

「**PNG画像を読み込み**」ボタンをクリックし、マップ画像（PNG形式）を選択します。

- 画像の読み込み後、画面左下にファイル名が表示されます
- 画像が表示されたら、各種操作ボタンが有効になります

### ステップ2: 編集モードを選ぶ

画面右上のコントロールパネルにあるラジオボタンで編集モードを切り替えます：

| モード | 何を記録するか |
|---|---|
| **ポイント編集** | 山頂・分岐点など特定の地点（赤い円） |
| **ルート編集** | 歩行ルートの経路（オレンジの菱形） |
| **スポット編集** | 観光地・休憩所など名前付きの場所（青い四角） |
| **エリア編集** | 危険箇所・植生保護区など面的な領域（ピンクの多角形） |

### ステップ3: 地図上にマーキングする

選択したモードに合わせて地図上をクリックしてマーキングします。

### ステップ4: データを保存する

コントロールパネルの「**保存**」ボタン（またはマーカー設定の「ファイル入出力」タブ）からJSONファイルに保存します。

## 3. コントロールパネルの移動（v6.2 新機能）

コントロールパネル上部にある**薄いベージュ色の帯**（タイトル「Points/Routes Marker」）が、新しいドラッグハンドルです。

### 移動方法

1. タイトル部分（薄いベージュの帯）にマウスカーソルを乗せます
2. カーソルが「移動カーソル（十字矢印）」に変わります
3. ドラッグして好きな位置に移動します
4. パネルは自動的にウィンドウ内に収まる範囲で移動します

> **ヒント**: 画像の右上を確認したいときや、編集対象がパネルに隠れているときに移動すると便利です。

---

## 4. 各編集モードの操作方法

### 4.1 ポイント編集

**ポイントの追加**
1. 「ポイント編集」モードを選択します
2. 地図上のマーキングしたい場所をクリックします
3. 赤い円が表示され、入力ボックスが現れます
4. ポイントID（例：`A-01`）を入力してEnterキーを押します

**ポイントIDのルール**
- 形式: アルファベット1文字 + ハイフン + 2桁数字（例：`A-01`、`B-12`）
- 全角文字・小文字は自動的に変換されます(例：`a-1` → `A-01`)
- 同じIDを重複して使用することはできません（赤枠で警告）

**ポイントの移動**
- ポイント上でドラッグして移動します

**ポイントの削除**
- 入力ボックスを空にしてEnterキーまたはフォーカスを外します
- または、Escapeキーを押します

---

### 4.2 ルート編集

**ルートの作成手順**
1. 「ルート編集」モードを選択します
2. 「**追加**」ボタンをクリックして新規ルートを作成します
3. 「開始ポイント」入力欄をクリックし、設定したいポイントID（例：`A-01`）を入力します
4. 「終了ポイント」入力欄も同様に設定します
5. 地図上をクリックして中間点を順番に配置します

**複数ルートの管理**
- ドロップダウンリストから編集したいルートを選択します
- 「削除」ボタンで選択中のルートを削除できます

**中間点の移動・削除**
- 中間点（オレンジの菱形）をドラッグして移動します
- 中間点を右クリックして削除します

**選択中のルートと非選択ルートの違い**
- 選択中のルート: 中間点が通常サイズで表示
- 非選択のルート: 中間点が小さく表示（邪魔にならないように）

---

### 4.3 スポット編集

**スポットの追加**
1. 「スポット編集」モードを選択します
2. 地図上のスポット場所をクリックします
3. 青い四角が表示され、入力ボックスが現れます
4. スポット名（例：「展望台」「休憩所」）を入力してEnterキーを押します

**スポットの移動・削除**
- スポット上でドラッグして移動します
- 入力ボックスを空にしてフォーカスを外すと削除されます

---

### 4.4 エリア編集

**エリアの作成**
1. 「エリア編集」モードを選択します
2. 「**追加**」ボタンをクリックして新規エリアを作成します
3. エリア名（例：「崩落箇所」「植生保護区」）を入力します
4. 地図上をクリックして頂点を3か所以上配置します
   - 3か所以上配置すると多角形がピンク色の線で描画されます
   - 頂点は自動的に整列され、きれいな多角形が保たれます

**エリアの修正**
- ピンク色の頂点マーカー（菱形）をドラッグして形状を調整します
- エリア内をクリックすると新しい頂点が追加されます
- ドロップダウンリストから編集したいエリアを切り替えます

> **ヒント**: 複雑な凹凸のある形状は表現できない場合があります。その場合は複数のエリアに分けて作成してください。

## 5. ズームと移動

コントロールパネル下部のボタンで地図のズームと移動ができます：

| ボタン | 操作 |
|---|---|
| ＋（ズームイン） | 地図を拡大（最大5倍まで） |
| −（ズームアウト） | 地図を縮小（元サイズまで） |
| ↑↓←→（矢印） | 地図を50pxずつ移動 |
| ↺（リセット・緑色の反時計回り矢印） | 最初の表示状態に戻す |

- ズーム・移動操作中も、入力ボックスの位置は自動的に追従します
- マーカーのサイズはズームレベルによらず画面上で一定に保たれます

### ⚠️ 重要：ズーム・移動中はデータ操作ができません（v6.2 新仕様）

画像をズームしたり上下左右に移動した状態では、次の操作が**一時的に無効**になります：

- DB読み込み・DB保存
- JSON出力
- ファイル読み込み・ファイル保存

これらのボタンをクリックすると、次の警告メッセージが表示されます：

```
画像がズームまたは移動された状態です。
ズームボタンの左にある「反時計回りの矢印」アイコン（表示リセット）で
元に戻してから実行してください
```

**対処方法**: ナビゲーションエリアにある「**表示リセットボタン**」（ズームボタンの左にある、緑色枠の反時計回り矢印アイコン）をクリックして表示を元に戻してから、再度操作してください。

## 6. データの保存と読み込み

### 6.1 JSONファイルへの保存（エクスポート）

**方法1: コントロールパネルの「出力」ボタン**（画像読み込み後に表示）

**方法2: 設定ダイアログ → 「ファイル入出力」タブ → 「💾 保存」**

どちらの方法でも、確認ダイアログは表示されず即座に保存対象選択ダイアログが開きます。

> **注意（v6.2）**: 画像をズーム・移動した状態では実行できません。先に表示リセットを行ってください。

**保存対象選択ダイアログ**:
- 保存したい要素（ポイント・ルート・スポット・エリア）を件数を確認しながら選択できます
- 0件の要素は自動的にチェックが外れます
- 「OK」を押すとファイル保存ダイアログが開きます

**保存ファイル名の形式**:
```
[画像名]_P[ポイント数]_R[ルート数]_S[スポット数]_A[エリア数]-[日付].json
例: trail_P10_R2_S5_A1-20260523.json
```

> **ヒント**: Chrome・Edge系ブラウザでは、保存先フォルダを自由に選択できます。デフォルトは画像ファイルと同じフォルダです。

### 6.2 JSONファイルからの読み込み（インポート）

**方法1: コントロールパネルの「読み込み」ボタン**（画像読み込み後に表示）

**方法2: 設定ダイアログ → 「ファイル入出力」タブ → 「📂 読み込み」**

確認ダイアログは表示されず即座にファイル選択ダイアログが開きます。

> **注意**: 読み込みを実行すると、現在のすべてのマーキングデータが消去されます。必要に応じて先に保存してください。
> 画像をズーム・移動した状態では実行できません。先に表示リセットを行ってください。

## 7. データベース連携（Firebase）

クラウドデータベース（Firebase）への保存・読み込みも可能です。

> **前提**: この機能を使用するには、別途Firebaseの設定が必要です。

### 7.1 DBから読み込む

コントロールパネルの「**読み込み**」ボタンをクリックします。

- 初回のみFirebaseへの接続・認証が行われます（画面右上に「接続中...」と表示）
- 読み込み中は再クリックしても「読み込み中です。完了までお待ちください」と表示され、処理は1回のみ実行されます

#### 進捗メッセージ（v6.2 新機能）

読み込み中、画面中央に現在の処理状況が表示されます：

```
ポイントのデータを読み込み中...
↓
ルートのデータを読み込み中...
↓
スポットのデータを読み込み中...
↓
エリアのデータを読み込み中...
```

> **注意**: 0件のデータ種別は表示も処理もスキップされます（例：エリアが0件ならエリアのメッセージは出ません）。

**完了時**には件数サマリーが3秒間表示されます：

```
読み込みが完了しました。
ポイント: 10件
ルート: 2件
スポット: 5件
エリア: 1件
```

### 7.2 DBに保存する

コントロールパネルの「**保存**」ボタンをクリックします。

- 保存中は再クリックしても「保存中です。完了までお待ちください」と表示され、二重保存は行われません

#### 進捗メッセージ（v6.2 新機能）

保存中も同様に進捗が表示されます：

```
ポイントのデータを保存中...
↓
ルートのデータを保存中...
↓
スポットのデータを保存中...
↓
エリアのデータを保存中...
```

**完了時**:

```
保存が完了しました。
ポイント: 10件
ルート: 2件
スポット: 5件
エリア: 1件
```

> **注意（v6.2）**: 画像をズーム・移動した状態では実行できません。先に表示リセットを行ってください。

## 8. マーカーサイズの設定

コントロールパネル下部の**設定ボタン（歯車アイコン）**をクリックすると、設定ダイアログが開きます。

**「マーカーサイズ設定」タブ**で各マーカーのサイズを調整できます：

| 設定項目 | 対象 | 範囲 | 初期値 |
|---|---|---|---|
| ポイント | 赤い円のサイズ | 2〜12px | 6px |
| 選択ルート中間点 | 編集中ルートの中間点（橙色）のサイズ | 2〜12px | 6px |
| 非選択ルート中間点 | 編集対象外ルートの中間点のサイズ | 2〜12px | 4px |
| スポット | 青い四角のサイズ | 4〜20px | 12px |
| エリア頂点 | ピンクの頂点マーカーのサイズ | 2〜12px | 6px |

- スライダーを動かすか、数値を直接入力してサイズを変更します
- 「**OK**」を押すと設定が保存され、次回起動時にも自動適用されます
- 「**初期値に戻す**」で全サイズをデフォルト値にリセットできます
- 「**キャンセル**」で変更を破棄します

## 9. 通知メッセージについて

操作の結果は画面中央に一時的なメッセージで表示されます：

| 色 | 意味 | 表示時間 |
|---|---|---|
| 水色 | 完了・情報（読み込み完了、保存完了など） | 3秒 |
| オレンジ | 警告（操作中の再クリック、ズーム中の操作拒否など） | 4.5秒 |
| 赤 | エラー | 6秒 |
| （進捗メッセージ・v6.2） | 処理中表示 | 処理完了まで表示し続ける |

## 10. よくある質問

**Q. ブラウザで直接ファイルを開いたら動作しない**
A. ローカルサーバーから開く必要があります。`python -m http.server 8000` を実行してから `http://localhost:8000` でアクセスしてください。

**Q. 保存ボタンを押してもファイルが保存されない**
A. まず画像を読み込んでください。画像読み込み後に保存ボタンが表示されます。
また、画像をズームまたは移動した状態では保存ができません（v6.2）。「表示リセット」ボタンで元に戻してから実行してください。

**Q. 「画像がズームまたは移動された状態です」と表示されて操作できない（v6.2）**
A. データ操作時の整合性確保のため、ズーム・移動中は読み込み・保存・出力が制限されます。ナビゲーションエリアにある緑色枠の反時計回り矢印（表示リセットボタン）を押して、画像を元の表示状態に戻してから再度操作してください。

**Q. コントロールパネルが邪魔で画像の右上が見えない（v6.2）**
A. パネル上部の薄いベージュ色のタイトル部分をドラッグして、別の場所に移動できます。

**Q. 進捗メッセージが消えないけど大丈夫？（v6.2）**
A. これは正常な動作です。データ処理中は進捗メッセージが表示され続け、処理が完了すると自動的に完了サマリーに切り替わります。

**Q. エリアの形がおかしい**
A. 頂点は自動的に重心から時計回り/反時計回りに整列されます。凹型の複雑な形状は表現できない場合があります。複数のエリアに分けて作成してください。

**Q. 読み込み中/保存中に再クリックしてしまった**
A. 警告メッセージが表示されますが、処理は1回だけ実行されます。そのままお待ちください。

**Q. ポイントIDが自動で変わってしまう**
A. 小文字は大文字に、1桁数字は2桁にゼロ埋めされる自動補正が働いています。`a-1` と入力すると `A-01` に変換されます。

**Q. 保存したJSONファイルを別の画像に読み込めるか**
A. 座標値は元画像のピクセルサイズ基準で保存されているため、別の画像に読み込むと位置がずれます。同じ画像で使用してください。

## 11. 推奨環境

| ブラウザ | 対応状況 |
|---|---|
| Chrome 86+ | 推奨（すべての機能が使用可能） |
| Edge 86+ | 推奨（すべての機能が使用可能） |
| Firefox（最新版） | 基本対応（File System Access API非対応のためファイル保存先選択不可） |
| Safari 14+ | 基本対応（File System Access API非対応のためファイル保存先選択不可） |

**ディスプレイ設定**: Windows拡大率125%・150%等の高DPI環境に対応しています。
